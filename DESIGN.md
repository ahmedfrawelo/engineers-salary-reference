# Design Context

## Scene

An engineer checks salary ranges on a laptop during a workday, comparing rows and filters quickly before submitting one report. The interface should be light, readable, and dense enough for repeated use.

## Theme

Product UI aligned with the ENGINEERS_SALARY_REFERENCE reference system: white and near-white light surfaces, gray dark mode surfaces, compact controls, and a restrained ENGINEERS_SALARY_REFERENCE green primary accent. Avoid decorative hero treatment, purple gradients, beige-only palettes, and dark blue dashboard styling.

## UI Rules

- Use cards only for individual panels and metrics.
- Keep form fields compact and aligned.
- Tables should be scrollable on small screens.
- Use stable dimensions for controls and metric blocks.
- Copy should be direct and operational.

## Palette

- Light page: `rgb(255 255 255)`
- Light shell: `rgb(249 249 249)`
- Light border: `rgb(232 232 232)`
- Dark page: `rgb(17 17 17)`
- Dark shell: `rgb(25 25 25)`
- Dark surface: `rgb(32 32 32)`
- Accent: `rgb(132 199 24)`
- Dark accent text: `rgb(190 242 100)`
