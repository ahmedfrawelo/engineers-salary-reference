# Quality Documentation

This folder mixes two document types:

- baseline snapshots: historical reference points
- generated reports: current state from scripts

## Files

- `baseline-clean-architecture.md`: historical baseline snapshot.
- `no-explicit-any-baseline.md`: historical explicit-any baseline.
- `complexity-report.md`: generated by `npm run complexity-report`.
- `no-explicit-any-report.md`: generated by `npm run type-safety:guard`.

## Refresh Current Reports

```bash
npm run complexity-report
npm run type-safety:guard
```

## Recommended Validation Flow

```bash
npm run architecture:check
npm run architecture:health
npm run complexity-report
npm run type-safety:guard
npm run verify:quick
```

## Reading Guidance

- Treat baseline files as historical context only.
- Treat generated reports as the source of truth for current quality status.
- If thresholds change in scripts, regenerate reports in the same PR.
